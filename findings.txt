Findings

This file records the current codebase facts and the saved experimental outputs present in the repository snapshot.

## 1. Task, models, and training setup

- Task: ISIC 2019 8-class skin lesion classification.
- Class labels: MEL, NV, BCC, AK, BKL, DF, VASC, SCC.
- Active scripts:
  - `train.py`
  - `test.py`
- Model backbones:
  - `baseline` uses `timm.create_model("vit_base_patch16_224", pretrained=False, num_classes=8)`
  - `swin` uses `timm.create_model("swin_tiny_patch4_window7_224", pretrained=False, num_classes=8)`
  - `foundation` uses zero-shot CLIP inference with `openai/clip-vit-large-patch14`
- Training policy:
  - baseline and swin are trained from scratch
  - techniques are only enabled for swin
  - foundation is test-only and does not use a checkpoint
- Default training hyperparameters in `train.py`:
  - epochs: 50
  - batch size: 64
  - learning rate: 1e-4
  - optimizer: AdamW
  - scheduler: CosineAnnealingLR
  - weight decay: 1e-4
  - seed: 42
  - save every: 10 epochs
  - AMP: disabled by default

## 2. Techniques implemented in the code

- Technique 1:
  - inserts `RulerCropTransform` and `SyntheticRulerAugmentation` before `ToTensor`
  - goal: suppress ruler dependence and add ruler-like augmentation
- Technique 2:
  - uses `AttentionRegularizationLoss(lambda_weight=0.1)`
  - goal: penalize attention on border patches
- Technique 3:
  - uses `BorderPatchMasker(mask_prob=0.5)`
  - goal: mask border patch embeddings during training
- Foundation prompts:
  - "a dermoscopic image of {}"
  - "a photo of {}"
  - "a skin lesion showing {}"
- Foundation class-name aliases:
  - MEL -> melanoma
  - NV -> melanocytic nevus
  - BCC -> basal cell carcinoma
  - AK -> actinic keratosis
  - BKL -> benign keratosis
  - DF -> dermatofibroma
  - VASC -> vascular lesion
  - SCC -> squamous cell carcinoma

## 3. Dataset and split facts

- Split generation uses `StratifiedShuffleSplit` with seed 42.
- The code removes `UNK` rows and keeps only samples with exactly one positive class label.
- Split ratio is 80/10/10 for train/val/test.
- `test_no_ruler.csv` and `test_with_ruler.csv` are created from the same `test_df`, so they are duplicates of the same test split.

## 7. Inference from the saved results

- The dataset is strongly imbalanced, especially for DF, VASC, SCC, and AK.
- The class-weighted loss is justified by the training distribution.
- Baseline is a scratch-trained reference, not a pretrained fine-tune.
- Swin without techniques is stronger than baseline on macro precision, recall, and F1.
- Technique 1 gives the best macro recall and macro F1 among the completed runs.
- Technique 2 is close to technique 1 on recall, but its macro F1 is lower.
- Technique 3 is the weakest of the Swin runs and drops sharply in accuracy.
- Foundation zero-shot CLIP is far below the trained models on this task.
- The `test_no_ruler` and `test_with_ruler` outputs are identical to the full test outputs because they are built from the same split. That makes the robustness tables a pipeline sanity check rather than a real ruler-bias comparison on this dataset.
- Baseline and Swin are compliant with the no-pretraining requirement.
- Foundation is inference-only, and the repo no longer writes a fake Grad-CAM image for it.

## 8. Important notes and limitations for the report

- The project is motivated by ruler bias in dermoscopic images, but the processed ISIC 2019 setup used here does not provide explicit ruler/no-ruler annotations.
- Because ruler annotations are unavailable, the project should be framed as ruler-like border artifact sensitivity and mitigation, not as a direct measurement of true ruler-present versus ruler-absent bias.
- The no-ruler and with-ruler test splits are duplicated from the same test split. Their identical metrics should be described as a pipeline sanity check, not as evidence that the model is robust to real rulers.
- The implemented techniques are still relevant to the ruler-bias motivation because they target border artifacts and ruler-like visual shortcuts:
  - Technique 1 uses border cropping plus synthetic ruler-like augmentation.
  - Technique 2 penalizes attention on border patches.
  - Technique 3 masks border patch embeddings during training.
- Adding an external ruler-image dataset at this stage is not recommended unless it is cleanly labeled with the same 8 class mapping: MEL, NV, BCC, AK, BKL, DF, VASC, SCC.
- A rushed external dataset could weaken the project because label mappings, class definitions, dataset shift, and ruler annotations would all need careful validation.
- The safer report framing is: "Because ISIC 2019 does not provide explicit ruler annotations in our processed split, we evaluate ruler-like artifact mitigation using proxy interventions and qualitative Grad-CAM/error analysis. This is a limitation of the study."
- Foundation does not need an epoch trend because it is zero-shot inference only and is not trained.
- Swin Technique 3 is missing the epoch 21-30 trend due to unrecoverable Colab logs. The report should state this transparently while noting that checkpoints and final evaluation artifacts were preserved.

### Report-ready main conclusion

- The baseline ViT trained from scratch is a valid reference model.
- Swin trained from scratch improves over the baseline on macro precision, macro recall, and macro F1.
- Technique 1 gives the best overall macro recall and macro F1 among the completed trained runs.
- Technique 2 improves recall relative to Swin none but has weaker macro F1 than Technique 1.
- Technique 3 underperforms the other Swin variants and appears to reduce overall classification quality.
- Zero-shot CLIP foundation inference performs much worse than the trained models, which is expected because it is not trained or fine-tuned on ISIC 2019.
- The study supports the usefulness of targeted border-artifact mitigation as a proxy method, but it cannot claim true ruler/no-ruler robustness without explicit ruler annotations.
